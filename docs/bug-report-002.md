# BUG-002: Граничный перевод (сумма+комиссия = доступный остаток) отклоняется

**ID:** BUG-002  
**Дата обнаружения:** 2026-05-23  
**Обнаружил:** Алексей Грейвс  
**Приложение:** F-Bank, localhost:8000  
**Тест-кейс:** TC-005  

---

## Серьёзность (Severity)

**Major** — приложение блокирует легитимный перевод: пользователь не может перевести ровно всю доступную сумму, хотя по условиям банка это разрешено.

## Приоритет (Priority)

**High**

---

## Описание

По спецификации перевод разрешён, если `сумма + комиссия ≤ доступный остаток`.  
В коде условие проверяется строгим неравенством:

```javascript
// в коде (неверно):
const allowed = balance - reserved - commission - amount > 0;

// по спецификации (верно):
const allowed = balance - reserved - commission - amount >= 0;
```

В результате граничный случай, когда `сумма + комиссия` в точности равна доступному остатку, обрабатывается как ошибка вместо разрешения операции.

---

## Шаги воспроизведения

1. Открыть http://localhost:8000/?**balance=1100**&**reserved=0**  
   (доступный остаток = 1100 − 0 = **1100 ₽**)
2. Кликнуть по блоку «Рубли»
3. Ввести любой 16-значный номер карты
4. В поле «Сумма перевода» ввести: **1000**  
   (комиссия = floor(1000 / 100) × 10 = **100 ₽**, итого = 1000 + 100 = **1100 ₽**)
5. Проверить, что отображается на экране

## Ожидаемый результат

Отображается кнопка **«Перевести»** — перевод должен быть разрешён, так как 1100 ≤ 1100.

## Фактический результат

Отображается сообщение об ошибке: **«Недостаточно средств на счете»** — перевод заблокирован.
