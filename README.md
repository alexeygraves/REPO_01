## Объект тестирования

**F-Bank** — фиктивный банковский веб-сервис для рублёвых переводов.

Запуск сервиса локально:
```bash
python3 -m http.server 8000 --directory dist
```

Открыть в браузере: `http://localhost:8000/?balance=30000&reserved=20001`

## Структура репозитория

```
dist/                      — статические файлы сервиса F-Bank
docs/
  test-cases.md            — 5 ручных тест-кейсов (TC-001..TC-005)
  bug-report-001.md        — BUG-001: неверная формула расчёта комиссии
  bug-report-002.md        — BUG-002: граничный перевод отклоняется вместо разрешения
tests/
  test_transfer.py         — 5 Selenium-тестов на pytest
  requirements.txt         — зависимости: selenium, pytest
.github/workflows/
  ci.yml                   — GitHub Actions CI
```

## Найденные дефекты

| ID | Описание | Severity |
|----|----------|----------|
| BUG-001 | Комиссия считается по формуле `floor(amount/100)*10` вместо `floor(amount/10)`. Для суммы 150 ₽ показывает 10 ₽ вместо 15 ₽. | Major |
| BUG-002 | Условие доступности использует `>` вместо `>=`. Перевод, где сумма+комиссия равна доступному остатку, отклоняется. | Major |

## Запуск тестов локально

```bash
pip install -r tests/requirements.txt
# сервис должен быть запущен:
python3 -m http.server 8000 --directory dist &
pytest tests/ -v
```

## CI

Сборка **намеренно красная**: тесты TC-004 и TC-005 воспроизводят реальные дефекты приложения.
